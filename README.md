# 世界杯预测系统 WorldCupPredictor

**版本** v1.0  
**日期** 2026-05  
**项目** WorldCup Predictor Web App

---

## 一、概述

### 1.1 背景

基于现有 Python 数据管道（fetch / features / model / simulate），构建一个可视化 Web 前端，让用户无需命令行即可使用全部预测功能。

### 1.2 核心能力

| 模块 | 文件 | 能力 |
|---|---|---|
| 数据拉取 | fetch.py / fetch_squads.py / fetch_external.py 等 | 赛事结果、阵容、市值、xG、天气、赔率、FIFA排名 |
| 特征工程 | features.py | 7类 60+ 特征：状态/H2H/阵容/战术/外部/Elo/衍生 |
| 模型训练 | model.py | LogReg / RF / XGBoost / LightGBM，时序交叉验证 |
| 预测模拟 | simulate.py | 泊松进球回归 + 蒙特卡洛赛程模拟 |
| 反事实 | simulate.py + historical_squads.py | 历史阵容参加当届世界杯 |

### 1.3 目标用户

- 足球爱好者：直观看各队夺冠概率
- 数据分析师：探索特征重要性、模型评估
- 内容创作者：生成分析内容素材

### 1.4 技术栈建议

```
前端:  React 18 + TypeScript + TailwindCSS + Recharts / D3.js
后端:  FastAPI (Python) — 直接复用现有模块
存储:  SQLite（从 parquet 迁移）+ 内存缓存
部署:  Docker Compose（前端 + 后端 + 定时任务）
```

## 三、页面详细需求

---

### 页面 1 — 首页 / 仪表盘

**目标**：一眼看懂当前世界杯格局，快速进入各功能。

#### 模块 1.1 赛事倒计时
- 距世界杯开幕的天/时/分倒计时
- 当前阶段标注（抽签期 / 小组赛 / 淘汰赛 / 决赛）

#### 模块 1.2 夺冠赔率热力图
- 32 支参赛队按夺冠概率排列的横向条形图
- 颜色按大洲区分（蓝=欧洲，绿=南美，橙=亚洲 等）
- 点击球队跳转球队详情页
- 数据来源标注（模型预测 vs 市场赔率 可切换）

#### 模块 1.3 近期热门比赛
- 未来7天的预定比赛卡片，每张显示：
  - 双方队徽 + 名称
  - 模型预测胜率（三格进度条）
  - 最可能比分
  - 点击进入完整预测页

#### 模块 1.4 数据健康状态
- 各数据源最后更新时间（绿=正常/黄=过期/红=失败）
- 数据源：football-data.org / StatsBomb / Transfermarkt / 天气 / 赔率

#### 模块 1.5 快速预测入口
- 两个下拉选队，点击「预测」跳转预测页

---

### 页面 2 — 比赛预测

**目标**：输入任意两支队，输出完整预测结果。

#### 模块 2.1 队伍选择器
- 两侧各一个搜索+下拉选队组件（支持模糊搜索）
- 中间显示「VS」
- 可选：比赛阶段（小组赛 / 淘汰赛 / 决赛）— 影响泊松λ

#### 模块 2.2 结果概率展示
- 三格横向进度条：主队胜% / 平% / 客队胜%
- 数字精确到小数点后一位
- 同时展示：分类模型概率 vs 泊松模型推导概率（可对比）

#### 模块 2.3 期望进球 & 比分矩阵
- 两队期望进球（λ_home / λ_away）大字显示
- N×N 比分热力图（0-0 到 5-5）
  - 颜色深浅 = 该比分概率
  - 鼠标悬停显示具体概率
  - 最高概率比分高亮边框
- Top 8 最可能比分列表（比分 + 概率 + 进度条）

#### 模块 2.4 淘汰赛专项（阶段=淘汰赛时出现）
- 含加时赛概率
- 含点球大战概率
- 双方历史点球胜率

#### 模块 2.5 关键特征解读
- 展示影响本场预测的 Top 5 特征（特征名 + 数值 + 对哪队有利）
- 例：「Elo 差值：+120（有利主队）」
- 用颜色区分正负影响

#### 模块 2.6 历史交锋摘要
- 总交锋次数 / 主队胜率 / 平局率 / 客队胜率
- 近5次交锋结果时间线（W/D/L 图标 + 比分）

---

### 页面 3 — 赛程模拟

**目标**：蒙特卡洛模拟整届世界杯，展示每队晋级概率。

#### 模块 3.1 模拟控制面板
- 模拟次数滑块：1万 / 5万 / 10万（影响精度和速度）
- 「开始模拟」按钮，带进度条（WebSocket 实时推送进度）
- 上次模拟时间 & 结果缓存提示

#### 模块 3.2 夺冠概率排行榜
- 32队按夺冠概率排序的表格
- 列：球队 / 出线% / 16强% / 8强% / 4强% / 决赛% / 冠军%
- 每行可展开查看该队的比赛路径分布

#### 模块 3.3 赛程流向图（Sankey / Bracket）
- 可视化淘汰赛对阵图
- 每条路径粗细 = 该队走这条路的概率
- 颜色 = 夺冠概率热度

#### 模块 3.4 小组赛出线概率
- 12个小组的网格排列
- 每格：4支队 + 出线概率条形图
- 点击小组展开详细对阵预测

#### 模块 3.5 对比视图
- 选择2~4支队，叠加对比它们在各轮的晋级概率曲线
- X轴：赛段；Y轴：概率

---

### 页面 4 — 球队详情

**目标**：单支球队的全面数据画像。

路由：`/team/Brazil`

#### 模块 4.1 基本信息卡
- 队徽 / 国旗 / 队名
- FIFA 排名 + 变动趋势（折线）
- 主教练姓名 + 执教年限 + 胜率
- 本届世界杯所在小组

#### 模块 4.2 近期状态
- 近10场比赛时间线（W/D/L 色块 + 对手 + 比分）
- 近5场 vs 近10场 关键指标对比（雷达图）
  - 进攻：进球 / xG / 射门
  - 防守：失球 / xGA / 零封
  - 状态：PPG / 连胜/连败

#### 模块 4.3 球员阵容
- 球员列表（可按位置/市值/出场数排序）
- 列：球员名 / 位置 / 俱乐部 / 市值 / 评分 / 伤停状态
- 球员点击展开近两赛季统计

#### 模块 4.4 大赛表现历史
- 历届世界杯成绩时间线（折线图：1930年至今最好成绩）
- 淘汰赛 vs 小组赛胜率对比
- 点球大战历史记录

#### 模块 4.5 战术指标（StatsBomb）
- xG / xGA / PPDA 近几届比赛均值
- 与本届其他参赛队的横向对比（散点图）

---

### 页面 5 — 历史对决 H2H

**目标**：任意两队历史交锋全记录。

#### 模块 5.1 选队
- 同预测页的双队选择器

#### 模块 5.2 总体统计
- 大数字：交锋次数 / A队胜 / 平 / B队胜
- 饼图展示胜负比例

#### 模块 5.3 比赛记录列表
- 时间 / 赛事 / 阶段 / 比分 / 主客 / 场馆
- 可筛选：仅世界杯 / 仅大赛 / 仅淘汰赛
- 点击比赛行展开详情（进球时间线 / 红黄牌）

#### 模块 5.4 时间趋势
- 折线图：按年代展示双方 Elo 差值变化
- 标注重大比赛节点（如1998决赛、2014决赛）

#### 模块 5.5 场馆 & 主客分析
- 在中立场地 vs 一方主场的胜率差异
- 气温/海拔对比赛结果的影响（散点图）

---

### 页面 6 — 反事实模拟（What-if）

**目标**：「如果历史最强阵容参加今届世界杯，会怎样？」

#### 模块 6.1 历史阵容选择
- 左侧：选一支参赛队（将被替换）
- 右侧：选择历史版本（下拉：2006巴西 / 1998法国 / 2010西班牙…）
- 或自定义：手动填写各项特征数值

#### 模块 6.2 阵容对比
- 当前阵容 vs 历史阵容的关键指标对比表
  - 总市值 / 平均年龄 / Elo / xG均值 / PPDA 等
  - 每项显示差值和优劣方向

#### 模块 6.3 单场预测对比
- 选择对手，并排展示：
  - 当前阵容预测结果
  - 历史阵容预测结果
  - 差异高亮

#### 模块 6.4 全程模拟对比
- 运行两次蒙特卡洛（当前 vs 历史），结果并排
- 夺冠概率变化：「+X% / -X%」大字突出
- 各轮晋级概率折线对比图

#### 模块 6.5 自定义场景编辑器
- 滑块调整任意特征（如「假设姆巴佩缺席，进攻xG降低20%」）
- 实时更新预测结果（防抖，500ms延迟）

---

### 页面 7 — 模型评估

**目标**：展示模型性能，建立用户信任。

#### 模块 7.1 交叉验证结果
- 四模型（LogReg / RF / XGBoost / LightGBM）对比表
- 列：模型名 / 准确率 / LogLoss / RPS
- 最优模型高亮
- RPS 基准线：随机=0.333，博彩市场≈0.200

#### 模块 7.2 特征重要性
- Top 25 特征横向条形图
- 颜色按特征类别（Form/H2H/Squad/xG/External/Elo）
- 可切换：全局重要性 vs 某类比赛（如淘汰赛）的重要性

#### 模块 7.3 预测校准曲线
- 可靠性图（Reliability Diagram）
  - X轴：预测概率；Y轴：实际胜率
  - 完美校准 = 对角线
- 三条线：主队胜 / 平 / 客队胜

#### 模块 7.4 混淆矩阵
- W/D/L 三分类混淆矩阵热力图
- 标注：哪类结果最难预测（通常是平局）

#### 模块 7.5 历史回测
- 选择某届世界杯（如2022），用之前数据训练，预测该届所有比赛
- 展示：预测正确率 / 盈亏曲线（按赔率模拟投注）

---

### 页面 8 — 数据管理（Admin）

**目标**：后台管理数据拉取状态，手动触发更新。

#### 模块 8.1 数据源状态面板
- 每个数据源一行：
  - 数据源名 / 最后更新时间 / 数据量（行数）/ 状态图标
  - 数据源：football-data / StatsBomb / Transfermarkt / Open-Meteo / Odds API / FIFA Rankings

#### 模块 8.2 手动触发更新
- 每个数据源的「立即更新」按钮
- 更新日志实时滚动（WebSocket）
- 预计耗时提示

#### 模块 8.3 特征重建
- 「重新构建特征矩阵」按钮（触发 features.py --build）
- 显示上次构建时间 + 耗时

#### 模块 8.4 模型重训
- 「重新训练模型」按钮（触发 model.py --train）
- 展示上次训练结果摘要

## 五、后端 API 设计

```
GET  /api/teams                          # 所有球队列表
GET  /api/team/{name}                    # 球队详情
GET  /api/predict?home=X&away=Y          # 单场预测
GET  /api/h2h?team1=X&team2=Y            # 历史交锋
POST /api/simulate/tournament            # 赛程模拟（body: {n, groups}）
POST /api/simulate/whatif                # 反事实模拟（body: {team, overrides, n}）
GET  /api/model/metrics                  # 模型评估指标
GET  /api/model/importance               # 特征重要性
GET  /api/admin/data/status              # 数据源状态
POST /api/admin/data/refresh/{source}    # 触发数据更新
WS   /ws/simulate                        # 蒙特卡洛进度推送
WS   /ws/data/refresh                    # 数据更新日志推送
```

## 七、非功能性需求

| 项目 | 要求 |
|---|---|
| 单场预测响应时间 | < 2 秒 |
| 蒙特卡洛 5万次耗时 | < 30 秒（后台异步，WebSocket推送） |
| 移动端适配 | 响应式，主要页面支持手机浏览 |
| 数据缓存 | 预测结果缓存 1 小时，蒙特卡洛结果缓存 24 小时 |
| 数据自动更新 | 每日 UTC 06:00 自动拉取最新数据 |
